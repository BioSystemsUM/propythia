ProPythia modules¶

sequence¶

A class used for reading sequences or change sequences. The main objective is to create sequence objects to calculate descriptors. The class allows to:

1)Read sequences from string or from uniprot ID (is also possible retrieve sequences fom txt with uniprot IDs).

2)Check if the protein sequence is a valid sequence.

3)Obtain a sized sequence of list of sequences, adding or cutting from both n and c terminals.

4)From one sequence generate list of subsequences based on sliding window approach, from specific aa, from the terminals or divide the sequence in parts.

class propythia.sequence.ReadSequence[source]¶

Bases: object

The ReadSequence class aims to read the input and transform it into a sequence that can be used to calculate Descriptors.

It accepts uniprotID, txt with uniprot ID and string with aminoacid sequence. The functions to read protein are based on the package pydpi. It allows to get sized sequences or a variety of subsequences from just one sequence.

get_sized_seq(sequences=[], n_terminal=10, c_terminal=10, terminal=0)[source]¶

cut or add aminoacids to obtain sequences with equal length.

Parameters

  • sequences – list containing protein sequences (string) or just a protein sequence

  • n_terminal – number of aa to consider in the n terminal (left side of sequence)

  • c_terminal – number of aa to consider in the c terminal (right side of sequence)

  • terminal – in case of need to add dummy aa and no terminal has already been chosen, it decides where to add ‘0’ to add to the right (consider N terminal), ‘-1’ to add to the left (consider C terminal), ‘2’ to add in the middle (N and C terminal) will be both present and repeated with dummy in middle

recursive_feature_elimination(cross_validation=False, estimator=SVC(C=1.0, break_ties=False, cache_size=200, class_weight=None, coef0=0.0, decision_function_shape='ovr', degree=3, gamma='scale', kernel='linear', max_iter=-1, probability=False, random_state=None, shrinking=True, tol=0.001, verbose=False), n_features_to_select=None, min_features_to_select=1, cv=None, scoring=None, n_jobs=None, step=1, verbose=0, scaler=StandardScaler(copy=True, with_mean=True, with_std=True))[source]¶

Given an external estimator that assigns weights to features (e.g., the coefficients of a linear model), recursive feature elimination (RFE) is to select features by recursively considering smaller and smaller sets of features. First, the estimator is trained on the initial set of features and the importance of each feature is obtained either through a coef_ attribute or through a feature_importances_ attribute. Then, the least important features are pruned from current set of features.That procedure is recursively repeated on the pruned set until the desired number of features to select is eventually reached. RFECV performs RFE in a cross-validation loop to find the optimal number of features.

Parameters

  • scaler – scaler function to use to datasets before apply univariate tests. It can be None or any function supported by SKlearn like StandardScaler()

  • cross_validation – if yes: RFECV . if not: RFE

  • estimator – estimator that assign wights to features

  • n_features_to_select – to RFE

  • min_features_to_select – to RFECV

  • cv – number of folds in cross validation

  • scoring – for RFECV

  • n_jobs –

  • step – If greater than or equal to 1, then step corresponds to the (integer) number of features to remove at each iteration. If within (0.0, 1.0), then step corresponds to the percentage (rounded down) of features to remove at each iteration

  • verbose –

Returns

rfe fit, rfe transformed, original dataset with features selected, columns names and features ranking
